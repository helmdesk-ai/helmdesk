# Contributing

Thanks for helping improve HelmDesk.

## Before you start

- Open an issue or discussion first for larger changes so the direction is clear.
- Keep pull requests focused and avoid unrelated refactors.
- Add or update tests when your change affects behavior.
- Update docs when the user-facing workflow changes.

## Development

- Follow the setup steps in [readme.md](readme.md).
- Run the relevant checks before submitting:
  - `./vendor/bin/pest`
  - `npm run build`
  - `vendor/bin/pint`
  - `npm run format:check`
